# Skill: Revisão

Revisa capítulo produzido. Verifica fatos, qualidade narrativa, conexão com o arco, padrão mínimo.

## Antes de começar

1. Ler `coordination.md` — entender tese, arco, onde o capítulo se encaixa
2. Ler `playbook.md` seção 2, Etapa 4
3. Ler o `outline.md` aprovado (o draft seguiu o outline?)
4. Ler o `research.md` (os dados conferem?)

## Três verificações

### Factual
- Toda afirmação com fonte está correta?
- Dados numéricos conferem com a pesquisa?
- Nomes, datas, grafias corretos?
- Citações fiéis ao original?

### Narrativa
- Abre com cena/imagem?
- Tem fio narrativo claro?
- Avança a tese do livro?
- Pontes de entrada e saída funcionam?
- Referências cruzadas fazem sentido?
- Sensibilidades respeitadas?

### Padrão
- 8.000-12.000 palavras (capítulo)?
- 1.500+ palavras (cada seção)?
- 15+ fontes citadas?
- 3+ citações diretas?
- 2+ referências cruzadas?
- Sem anti-padrões (resumão, lista, clichê, texto genérico)?

## Formato do feedback

Para cada problema:
- **Onde**: trecho ou seção
- **O quê**: o problema
- **Como resolver**: sugestão concreta

Separar em:
1. **Problemas graves** — precisam ser corrigidos
2. **Melhorias** — sugestões para elevar qualidade
3. **Pontos fortes** — o que manter

## Veredito

- **Aprovado** → vai para o autor
- **Ajustes** → lista específica → volta para redação
- **Reescrever** → explica por quê → volta para outline

## Saída

`drafts/cap-XX-titulo/review.md`
